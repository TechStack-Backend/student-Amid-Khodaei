## سلام

به طور کلی وقتی ما داخل مرورگر خود یک URL وارد می‌کنیم (با فرض درست بودن اون URL) یک ریکویست برای سرور جنگو فرستاده میشه که این ریکویست از طریق وب سرور به برنامه جنگو فرستاد می‌شود.

و به طور کلی URL ها شامل بخش های مختلفی هستن

1_ protocol

2_domain

3_path

..

که در اینجا اون path نشون میده ریکویست چه چیزی میخواد و اون به طور خلاصه برای جنگو ارسال می‌شود.

حالا urls ما میاد اون path رو میخونه و میفهمه باید برا کدام بخش view بفرسته.
در قسمت بعد, بخش view اجرا میشه و چیزهایی که قراره برای ان مرورگر بفرسته رو به urls برمیگردونه که این میتونه هر چیزی که مرورگر توایی نمایششو داره باشه از عکس تا یک متن که توسط html نوشته شده یا ی سری دیتا که از model میگیره.

همچنین اگر نیازی به صفحه html بود template وارد کار می‌شود و آنرا تولید می‌کند.

و در نهایت view یا template ان پاسخی که ساختن را برای مرورگر به صورت respond http میفرستد.

در این بین هم ممکن است وب سرور در هنگام وصل شدن به ان مرورگر(client) رمزگذاری های لازم را انجام دهد ولی در جایی که جنگو کار میکند رمزگذاری در کار نیست.
